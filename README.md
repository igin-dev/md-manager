# md-manager

Система управления CLAUDE.md и другими MD-файлами для проектов на Claude Code.

5 slash-команд + 3 субагента для полного цикла:
создание → аудит → исправление → поддержка → оптимизация.

## Установка

```powershell
# Windows
git clone https://github.com/igin-dev/md-manager
cd md-manager
.\install.ps1 C:\path\to\your\project
```

```bash
# Linux / macOS
git clone https://github.com/igin-dev/md-manager
cd md-manager
chmod +x install.sh
./install.sh /path/to/your/project
```

Скрипт копирует `.claude/commands/`, `.claude/agents/` (включая rules и examples) в целевой проект. Существующие файлы не перезаписываются — для обновления добавь `--force`.

## Команды

| Команда | Когда использовать |
|---|---|
| `/md-init` | Новый проект — нет CLAUDE.md |
| `/md-audit` | Проверить качество существующего файла |
| `/md-fix` | Исправить проблемы найденные аудитом |
| `/md-sync` | После изменений в проекте (новый MCP, стек, gotcha) |
| `/md-optimize` | Файл вырос или появилось дублирование |

## Auto-triggering

Команды можно вызывать не только через `/slash`, но и естественным языком — Claude автоматически определит нужный скилл по описанию:

```
"Создай CLAUDE.md для проекта"        → md-init
"Проверь мой CLAUDE.md"               → md-audit
"Исправь проблемы в CLAUDE.md"        → md-fix
"Обнови CLAUDE.md" / "Обнови доку"    → md-sync
"CLAUDE.md слишком большой"            → md-optimize
```

## Типичный рабочий процесс

```
# Новый проект
/md-init                     → создаёт CLAUDE.md из контекста проекта

# Проверка и исправление
/md-audit hub/CLAUDE.md      → отчёт с кодами проблем по severity
/md-fix hub/CLAUDE.md        → исправляет найденные проблемы

# После изменений в проекте
/md-sync добавил MCP X       → показывает diff и применяет сразу

# Файл разросся
/md-optimize CLAUDE.md       → предлагает план разбивки на referenced файлы
```

## Архитектура

**Slash-команды** (`.claude/commands/`) — точки входа. Собирают контекст проекта, вызывают субагенты через Agent tool, управляют записью файлов.

**Субагенты** (`.claude/agents/`) — специализированная логика, изолированная от контекста сессии:
- `md-auditor` — чеклист встроен в промпт (source of truth для кодов: RED, SIZE, NAV, EXT, GOT, QUA)
- `md-creator` — читает rules/ и examples/ через Read tool
- `md-optimizer` — критерии оптимизации встроены в промпт

Модель субагентов наследуется от родительской сессии — работаете на Opus, субагенты тоже на Opus.

**Rules/** — справочник для md-creator и людей, не для аудитора. Коды аудита определены только в `md-auditor.md`.

## Подтверждения

Не на каждом шагу. Баланс между информированностью и скоростью:

| Команда | Подтверждение |
|---|---|
| `/md-init` | 1 — черновик перед записью |
| `/md-audit` | 0 — только вывод отчёта |
| `/md-fix` | 1 — показать правки, ждать "да" |
| `/md-sync` | 0 — показать diff и применить |
| `/md-optimize` | 1 — план перед выполнением |

## Структура

```
md-manager/
  .claude/
    commands/                # 5 slash-команд (точки входа)
    agents/
      md-auditor.md          # проверяет по встроенному чеклисту
      md-creator.md          # создаёт новые MD-файлы
      md-optimizer.md        # планирует оптимизацию
      rules/                 # 8 файлов — справочник для md-creator
      examples/              # 4 примера реальных хороших файлов
  install.ps1                # установка Windows (PS 5.1+, UTF-8 BOM)
  install.sh                 # установка Linux / macOS
```

## Монорепо

Поддерживается иерархия: корень → субпроект → инструмент. Каждый уровень наследует контекст родительского CLAUDE.md. Дочерний файл содержит только дельту — дублирование с родителем выявляется аудитом (RED-2).

## Главный принцип

**В CLAUDE.md должно быть только то, что агент НЕ МОЖЕТ вывести из кода сам.**

Это единственный критерий. Стандартные команды запуска, форматы логов, паттерны языка — агент выведет из кода. Gotchas, запреты с причиной, внешние автозапуски, MCP-серверы — нет.
