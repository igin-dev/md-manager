# Рекомендации: размер и плотность

## Главный критерий

Размер файла — не проблема сам по себе.
Проблема — когда файл большой из-за очевидного из кода, а не из-за реальных gotchas.

Маленький файл с водой хуже большого файла с реальными gotchas.
Большой файл с очевидным контентом хуже маленького файла только с неочевидным.

## Что считать избыточным

Контент очевидный из кода — агент выведет сам:
- Стандартные команды запуска, форматы логов, стандартные паттерны
- Для каждого такого места спроси: откуда агент это возьмёт?

"Вода" — слова без информации:
- Вступления: "В этом файле описано...", "Важно понимать что..."
- Повторение одного и того же разными словами
- Общие слова: "используй лучшие практики", "следуй стандартам"

## Когда выносить в отдельные файлы

Детальные справочные секции (больше 20 строк) → @CONVENTIONS.md или @ARCHITECTURE.md.
Применимо только если секция справочная (не gotcha, не запрет).

## Ориентиры по размеру

| Тип | Хорошо | Допустимо |
|---|---|---|
| root (монорепо) | 15–40 строк | до 60 |
| subproject | 30–60 строк | до 100 |
| tool (со сложными gotchas) | 50–120 строк | до 200 |

Превышение допустимого — повод проверить на очевидность контента, не автоматически проблема.
