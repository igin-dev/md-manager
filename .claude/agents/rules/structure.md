# Рекомендации по структуре MD-файлов

## Нет обязательных секций

Включай только те секции которые содержат неочевидную для агента информацию.
Пустые или очевидные секции хуже их отсутствия — раздувают контекст.

Главный тест: "Агент выведет это из кода сам?" Если да — не включай секцию.

## Типичные секции (включай по необходимости)

- **Навигация / Субпроекты** — только для монорепо. Агент не знает что нужно читать дочерний CLAUDE.md.
- **Автозапуск** — Task Scheduler, cron, systemd. Не видно из кода.
- **MCP-серверы** — если настроены. Агент не знает о них без указания.
- **Запуск** — только если нестандартный или важен контекст (не `python app.py`).
- **Gotchas** — неочевидное поведение с примером и причиной.
- **Запреты** — только с объяснением причины.
- **Ссылки** — @ARCHITECTURE.md, @CONVENTIONS.md если вынесены.

## Карта субпроектов (для root в монорепо)

Агент не знает о существовании дочерних CLAUDE.md без явной карты:

```markdown
| Директория | Описание | CLAUDE.md |
|---|---|---|
| `hub/` | Flask dashboard :5000 | `hub/CLAUDE.md` |
```

## Секция "Запуск" — когда включать

Включай ТОЛЬКО если:
- Нестандартная команда запуска
- Несколько способов запуска (таблица сценариев)
- Компонент запущен автоматически и повторный запуск опасен

НЕ включай: `python app.py`, `npm start`, `flask run` — агент выведет сам.

## Секция "Стек" — когда включать

Только то что влияет на решения агента:
- Библиотеки с нестандартным использованием
- Запрещённые альтернативы ("только TA-Lib, не VectorBT built-in")

НЕ включай: стандартные библиотеки без ограничений, всё что есть в requirements.txt без специфики.
