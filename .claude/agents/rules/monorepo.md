# Правила: монорепо и иерархия

## Чеклист

- [MONO-1] Корневой файл описывает глобальные coding standards **(info)**
- [MONO-2] Субпроект содержит только дельту от родителя **(warning)**
- [MONO-3] Субпроект явно ссылается на родителя если что-то наследует **(info)**

## Принцип иерархии

Claude Code читает CLAUDE.md снизу вверх по дереву директорий.
Каждый уровень наследует контекст родителя автоматически.

Это значит:
- Если глобальный coding standard написан в корневом CLAUDE.md — не повторяй в дочернем
- Если gotcha специфична только для одного инструмента — пиши только в его CLAUDE.md
- Если что-то одновременно в родителе и потомке — это дублирование

## Что должно быть на каком уровне

**Корневой CLAUDE.md (root):**
- Глобальные coding standards (логирование, обработка ошибок, комментарии)
- Карта субпроектов
- Глобальные MCP-серверы
- Общие запреты для всего монорепо

**Субпроект / инструмент (subproject / tool):**
- Только то что отличается от родителя или добавляется к нему
- Специфичные команды запуска
- Gotchas специфичные для этого инструмента
- MCP-серверы специфичные для этого инструмента

## Признаки дублирования

Дублирование — если в дочернем файле повторяется:
- Тот же coding standard что уже есть в родителе
- То же описание MCP-сервера
- Те же глобальные запреты

Исключение: если дочерний файл добавляет детали или уточняет правило
родителя для своего контекста — это не дублирование, это уточнение.

## Формат карты субпроектов (для root)

```markdown
## Структура

| Директория | Описание | CLAUDE.md |
|---|---|---|
| `hub/` | Flask dashboard :5000 | `hub/CLAUDE.md` |
| `hub/tools/telegram-analyzer/` | Telegram news analyzer | отдельный |
| `hub/tools/voice-input/` | Whisper voice input | отдельный |
```
