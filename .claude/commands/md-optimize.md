# /md-optimize

Запускается когда файл вырос, содержит дублирование или стал тяжёлым для чтения.

## Использование

```
/md-optimize                                    # все MD-файлы проекта
/md-optimize CLAUDE.md                          # конкретный файл
/md-optimize hub/tools/telegram-analyzer/       # директория
```

## Выполнение

**Шаг 1 — диагностика.**
Для каждого найденного файла собери:
- Количество строк
- Есть ли секции >40 строк без @-ссылок
- Есть ли дублирование с родительским CLAUDE.md
- Есть ли @-ссылки на несуществующие файлы

Файлы ≤150 строк — пропускай, они не нуждаются в оптимизации. Сообщи об этом.

**Шаг 2 — запустить субагент md-optimizer.**
Для каждого файла требующего оптимизации запусти Agent:

```
Ты — md-optimizer. Прочитай свои инструкции из .claude/agents/md-optimizer.md.

Файл для оптимизации ({N} строк):
{содержимое}

Родительский CLAUDE.md:
{содержимое или "отсутствует"}

Структура директории:
{список файлов}
```

**Шаг 3 — показать план и ждать подтверждения.**
Выведи план оптимизации от субагента.
Спроси: "Выполнить этот план? (можно указать номера пунктов)"

**Шаг 4 — выполнить и создать новые файлы.**
После подтверждения:
- Для каждого нового файла (CONVENTIONS.md и т.д.) — вызвать `/md-init`
- Показать итоговые версии всех изменённых файлов
- Записать только после финального "да"
