# /md-fix

Исправляет конкретные проблемы в MD-файле.

## Использование

```
/md-fix CLAUDE.md              # исправить все critical-проблемы
/md-fix CLAUDE.md GOT-1 STR-2  # исправить конкретные пункты по коду
/md-fix hub/CLAUDE.md          # файл по пути
```

## Выполнение

**Шаг 1 — разобрать аргументы.**

Разбери `$ARGUMENTS`:
- Только файл → исправить все critical-проблемы (запустить аудит сначала)
- Файл + коды (GOT-1, STR-2 и т.д.) → исправить только указанные проблемы

**Шаг 2 — запустить аудит.**

Прочитай файл через Read. Определи тип (root / subproject / tool).
Прочитай родительские CLAUDE.md через Read (для проверки дублирования).
Получи структуру директории через Glob.

Вызови субагент `md-auditor` через Agent tool.
Передай полное содержимое файла, родительские файлы и структуру директории как в `/md-audit`.

Если в аргументах указаны конкретные коды — работай только с ними.
Если нет — работай с найденными critical-проблемами.

**Шаг 3 — подготовить правки.**

Для каждой проблемы которую нужно исправить:
- Определи точное место в файле
- Покажи конкретное изменение в формате:

```
ИСПРАВЛЕНИЕ [GOT-1]: Добавить секцию "Запреты"
────────────────────────────────────────────
+ ## Запреты
+
+ **asyncio.run() — запрещено.** Закрывает общий event loop Telethon.
+ Используй только `_run_async(coro)` из `async_bridge.py`.
```

**Шаг 4 — запросить подтверждение.**

Покажи все запланированные правки.
Спроси: "Применить {N} исправлений к `{файл}`?"
Жди явного "да" перед записью.

**Шаг 5 — записать и проверить.**

Запиши обновлённый файл через Write.
Запусти повторный аудит через Agent tool (`md-auditor`) чтобы подтвердить что проблемы устранены.

$ARGUMENTS
