# Пример: CLAUDE.md для Flask Blueprint инструмента (hub)

Источник: реальный проект, hub/CLAUDE.md
Тип: subproject (субпроект монорепо)
Размер: 2.5 KB — хороший баланс

## Что здесь хорошо

- Одна строка описания в начале — сразу понятно что это
- Таблица запуска с указанием когда нужен перезапуск, а когда нет
- Чёткая типология модулей (EXTERNAL vs BLUEPRINT) с поведенческим описанием
- Пошаговая инструкция добавления нового инструмента — конкретная, без лишних слов
- Gotchas с объяснением механизма (не просто "осторожно с процессами")
- Ссылка на другой CLAUDE.md для деталей вместо копирования

```markdown
# Hub — центральная панель управления

Flask-приложение на порту 5000. Управляет инструментами (tools).

## Запуск

| Сценарий | Команда |
|---|---|
| Первый запуск / открыть браузер | Ярлык на рабочем столе (`start_hub.pyw`) |
| После изменений кода Hub | `restart_hub.bat` (двойной клик) |
| Из терминала (с выводом) | `python app.py` |

Нужен после изменений: `app.py`, `routes.py`, `modules/*.py`, `templates/`, `config.py`.
Не нужен: изменения в `voice_input.py` (отдельный процесс) и сохранение настроек через UI.

## Типы модулей

- `EXTERNAL` — отдельный процесс; Hub запускает/останавливает, статус через `status.json` (`loading`/`ready`/`error`).
- `BLUEPRINT` — Flask Blueprint, встроен в Hub, всегда запущен, общая авторизация.

## Добавить новый инструмент

1. Создать папку `hub/tools/<n>/`
2. Создать `hub/modules/<n>.py` с `ModuleInfo`
3. Добавить одну строку импорта в `hub/modules/__init__.py`

## Архитектурные подводные камни

**Дедупликация EXTERNAL-процессов**: `start_process()` убивает все существующие
экземпляры скрипта перед запуском (матч только по `python/pythonw` + `argv[1]`).
`detect_running()` при дубликатах выбирает процесс с наибольшим RSS.

**Два SQLAlchemy**: Hub и telegram-analyzer делят Flask-приложение.
Детали — в `hub/tools/telegram-analyzer/CLAUDE.md`.
```
