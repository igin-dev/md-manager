# /md-optimize

Запускается когда файл вырос, содержит дублирование или стал тяжёлым.

## Использование

```
/md-optimize                                          # все MD-файлы проекта
/md-optimize CLAUDE.md                                # конкретный файл
/md-optimize hub/tools/telegram-analyzer/CLAUDE.md   # по пути
```

## Выполнение

**Шаг 1 — диагностика.**

Найди все целевые файлы через Glob. Для каждого:
- Прочитай через Read, посчитай строки
- Проверь есть ли секции >40 строк без @-ссылок (кандидаты на вынос)
- Прочитай родительский CLAUDE.md — есть ли дублирование между уровнями
- Найди через Glob все @-referenced файлы — существуют ли они

Файлы ≤150 строк — пропусти, сообщи об этом явно.

**Шаг 2 — вызови субагент `md-optimizer`.**

Используй Agent tool.
Передай в prompt:

```
Проанализируй MD-файл и составь план оптимизации.

Файл: {имя} ({N} строк)

=== СОДЕРЖИМОЕ ===
{полное содержимое}

=== РОДИТЕЛЬСКИЙ CLAUDE.MD ===
{содержимое или "отсутствует"}

=== СТРУКТУРА ДИРЕКТОРИИ ===
{список файлов из Glob}
```

**Шаг 3 — показать план и ждать подтверждения.**

Выведи план от субагента. Спроси:
"Выполнить этот план? (можно указать номера пунктов, например: 1 3)"

**Шаг 4 — выполнить.**

После подтверждения:
- Для новых файлов (CONVENTIONS.md и т.д.) — вызови субагент `md-creator`
  через Agent tool с контекстом проекта
- Запиши обновлённый основной файл через Write
- Выведи итог: было X строк → стало Y строк
