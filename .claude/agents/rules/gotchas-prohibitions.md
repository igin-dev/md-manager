# Рекомендации: запреты и gotchas

## Что такое хороший запрет

Запрет без причины — бесполезен. Агент может "решить" что правило устарело.

Плохо:
```
Не использовать asyncio.run().
```

Хорошо:
```
asyncio.run() — ЗАПРЕЩЕНО. Закрывает общий event loop Telethon.
Используй только _run_async(coro) из async_bridge.py.
```

## Что такое хороший gotcha

Gotcha — информация которую невозможно вывести из кода, но которая критична.

Признаки хорошего gotcha:
- Описывает неочевидное поведение (библиотеки, ОС, API)
- Объясняет ПОЧЕМУ нельзя делать иначе
- Содержит пример правильного и/или неправильного кода

Плохо:
```
Будь осторожен с датами в SQLite.
```

Хорошо:
```
SQLite и даты: SQLAlchemy сохраняет datetime с пробелом (`2026-02-23 14:30:00`).
В raw SQL никогда не использовать `.isoformat()` — T-формат ломает сортировку.

# Правильно
cur.execute("WHERE date >= ?", (dt.strftime('%Y-%m-%d %H:%M:%S'),))
# Неправильно — T-разделитель!
cur.execute("WHERE date >= ?", (dt.isoformat(),))
```

## MCP-серверы

Если в проекте настроены MCP-серверы — должны быть перечислены с указанием
для чего использовать и какие конкретные инструменты доступны.

Хорошо:
```
## MCP-серверы

### Crypto.com Exchange MCP
- `get_candlestick` — OHLCV свечи для бэктестинга
- `get_ticker` — текущие цены
- `get_instruments` — список торговых пар
Формат символов: `BTC_USDT` (через подчёркивание).
```
