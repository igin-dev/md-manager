# /md-sync

Синхронизирует CLAUDE.md с актуальным состоянием проекта.

## Использование

```
/md-sync                                         # описать изменения в диалоге
/md-sync добавил MCP-сервер Crypto.com           # описание сразу
/md-sync hub/CLAUDE.md новый тип модуля HYBRID   # конкретный файл + изменение
```

## Выполнение

**Шаг 1 — понять что изменилось.**
Разбери `$ARGUMENTS`:
- Есть описание изменения → используй его
- Пусто → спроси: "Что изменилось в проекте?"

Типы изменений: новый MCP, смена библиотеки, новый gotcha, новый паттерн,
новый инструмент в монорепо, изменение соглашений по коду.

**Шаг 2 — определить затронутый уровень.**
- Изменение касается всего монорепо → корневой CLAUDE.md
- Изменение касается одного инструмента → только его CLAUDE.md
- Изменение на нескольких уровнях → обновить каждый, не дублировать

**Шаг 3 — собрать контекст.**
- Прочитай целевой CLAUDE.md
- Выполни `git diff --stat HEAD~5` — понять что реально менялось
- Если есть `.mcp.json` — сравни с тем что указано в CLAUDE.md

**Шаг 4 — подготовить diff через Agent.**
Запусти Agent с `subagent_type: "general-purpose"`:

```
Ты — эксперт по CLAUDE.md для Claude Code.

Текущий CLAUDE.md:
{содержимое}

Что изменилось в проекте:
{описание изменения}

Git-активность за последнее время:
{git diff --stat}

Задача: предложи конкретные изменения в файл в формате:

ДОБАВИТЬ в секцию "{секция}":
---
{новый текст}
---

ЗАМЕНИТЬ в секции "{секция}":
Было: {старый текст}
Стало: {новый текст}

УДАЛИТЬ из секции "{секция}":
{удаляемый текст}

Обоснование каждого изменения: {почему}
```

**Шаг 5 — применить только после подтверждения.**
Покажи предложенные изменения.
Спроси: "Применить?" — вноси только после явного "да".
