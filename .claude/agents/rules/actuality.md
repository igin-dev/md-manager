# Рекомендации: актуальность контента

## Принцип

Каждый факт в MD-файле — утверждение о текущем состоянии проекта.
Устаревший факт хуже отсутствия — агент будет действовать по неверной информации.

## Что проверять при написании

**Пути к файлам и скриптам** — существуют ли физически:
- Скрипты запуска: `restart_hub.bat`, `start_hub.pyw`
- Конфигурационные файлы: `voice_config.json`, `ai_config.json`
- Директории модулей: `hub/modules/`, `hub/tools/`

**@-ссылки** — referenced файлы должны существовать:
- `@ARCHITECTURE.md`, `@CONVENTIONS.md`
- Несуществующая ссылка → агент получит ошибку при попытке прочитать

**Имена в коде** — должны совпадать с реальными:
- Ключи настроек и БД
- Имена переменных среды (.env) → `os.getenv()` вызовы
- Имена MCP-инструментов → .mcp.json

## При обновлении (md-sync)

Перед добавлением нового контента — проверь что старый ещё актуален.
Лучше обновить два места за раз чем создать рассинхронизацию.
