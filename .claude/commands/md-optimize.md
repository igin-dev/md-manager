# /md-optimize — оптимизация и реструктуризация MD-файлов

Запускается когда файл вырос, стал слишком большим или содержит дублирование.
Предлагает план разбивки на referenced файлы, убирает избыточность.

## Использование

```
/md-optimize                         # оптимизировать все MD-файлы проекта
/md-optimize CLAUDE.md               # конкретный файл
/md-optimize hub/tools/telegram-analyzer/  # директория с файлами
```

## Инструкции агенту

**Шаг 1 — диагностика.**
Для каждого файла собери метрики:
- Количество строк
- Есть ли секции >50 строк без @-ссылок
- Есть ли дублирование с родительским CLAUDE.md
- Есть ли @-ссылки на несуществующие файлы

Файлы до 150 строк — пропусти, они не нуждаются в оптимизации.

**Шаг 2 — предложить план.**
Выведи план оптимизации ПЕРЕД любыми действиями:

```
Предлагаемые изменения для telegram-analyzer/CLAUDE.md (5.7 KB, 187 строк):

1. ВЫНЕСТИ в CONVENTIONS.md (новый файл):
   - Секция "Timezone и аналитические периоды" (31 строка)
   - Секция "Ключевые settings" (8 строк)
   Добавить в CLAUDE.md ссылку: @CONVENTIONS.md

2. УБРАТЬ дублирование с hub/CLAUDE.md:
   - Описание asyncio-bridge (уже есть в родителе)

3. СЖАТЬ секцию "Режимы работы":
   - 3 команды → таблица (экономия: ~8 строк)

Итого: 187 → ~110 строк (-41%)
```

**Шаг 3 — запросить подтверждение.**
Спроси: "Выполнить этот план?" Можно указать номера пунктов для частичного выполнения.

**Шаг 4 — выполнить через субагент md-optimizer.**
Передай план и содержимое файлов субагенту.
Для новых файлов (CONVENTIONS.md и т.д.) — использовать md-creator.

**Шаг 5 — показать результат перед записью.**
Покажи итоговые версии всех изменённых файлов. Записать только после "да".
