# /md-audit

Проверяет MD-файлы проекта на соответствие best practice.

## Использование

```
/md-audit                    # все CLAUDE.md в проекте
/md-audit CLAUDE.md          # конкретный файл
/md-audit hub/CLAUDE.md      # файл по пути
/md-audit hub/               # все MD-файлы в поддиректории
```

## Выполнение

**Шаг 1 — определить область аудита.**

Разбери `$ARGUMENTS`:
- Пусто → найди все `CLAUDE.md` через Glob `**/CLAUDE.md`, исключи `node_modules/`, `.venv/`
- Путь к файлу → работай только с ним
- Путь к директории → найди все `*.md` внутри кроме README

**Шаг 2 — собрать контекст для каждого файла.**

Для каждого найденного файла:
1. Прочитай файл через Read, посчитай строки
2. Определи тип:
   - `root` — нет CLAUDE.md в родительской директории
   - `subproject` — есть родительский CLAUDE.md на один уровень выше
   - `tool` — два и более уровней вложенности
3. Прочитай родительские CLAUDE.md вверх до корня (через Read)
4. Получи список файлов в директории через Glob

**Шаг 3 — вызови субагент `md-auditor`.**

Используй Agent tool.
Передай в prompt:

```
Выполни аудит MD-файла по чеклисту.

Файл: {имя файла}
Тип: {root|subproject|tool}
Строк: {количество}

=== СОДЕРЖИМОЕ ФАЙЛА ===
{полное содержимое}

=== РОДИТЕЛЬСКИЙ CLAUDE.MD ===
{содержимое или "отсутствует"}

=== СТРУКТУРА ДИРЕКТОРИИ ===
{список файлов из Glob}
```

**Шаг 4 — вывести результат и предложить следующий шаг.**

Выведи отчёт субагента без изменений.
Если найдены critical-проблемы — предложи: `/md-fix {файл}` для исправления.
Если файлов несколько — разделяй отчёты горизонтальной линией `---`.

$ARGUMENTS
