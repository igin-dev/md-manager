# Правила: качество записи

## Чеклист

- [QUA-1] Императивный стиль: «Используй X», «Не делай Y» **(warning)**
- [QUA-2] Нет капслока кроме максимум 1-2 намеренных акцентов **(info)**
- [QUA-3] Язык однороден — русский или английский, не вперемешку **(info)**
- [QUA-4] Нет восклицательных знаков **(info)**

## Императивный стиль (QUA-1)

CLAUDE.md читает агент, не человек. Агент следует инструкциям.
Слова "рекомендуется", "лучше", "желательно", "можно" — оставляют агенту пространство
для интерпретации. Императив убирает неоднозначность.

| Плохо | Хорошо |
|---|---|
| Рекомендуется использовать TA-Lib | Используй только TA-Lib для индикаторов |
| Лучше добавлять обработку ошибок | Оборачивай в try/except + logger.error |
| Можно использовать asyncio.run() | asyncio.run() запрещён — см. раздел Запреты |
| Следует учитывать timezone | Все datetime в БД — naive UTC. Конвертируй через ta_timezone.py |

## Капслок (QUA-2)

Допустимо: одно-два слова для критичного акцента (ЗАПРЕЩЕНО, НИКОГДА, КРИТИЧНО).
Недопустимо: целые предложения, секции, заголовки в капслоке.

Если встречается >3 слова в капслоке — это признак агрессивного тона,
который снижает читаемость и не улучшает следование инструкциям.

## Однородность языка (QUA-3)

- Технические термины на английском внутри русского текста — нормально
  (`asyncio.run()`, `Blueprint`, `RotatingFileHandler`)
- Целые предложения или секции на разных языках — проблема

Допустимо:
```
Используй только `TA-Lib` для индикаторов. Никогда `vbt.MA.run()`.
```

Проблема:
```
# Запуск (русский заголовок)
Run with: python app.py (английское содержимое)
```

## Восклицательные знаки (QUA-4)

Восклицательные знаки в инструкциях для агента — лишние.
Агент не реагирует на эмоциональную интонацию, только на содержание.
Если что-то важно — скажи это императивом и добавь причину.
