# Правила: актуальность

## Чеклист

- [ACT-1] Все пути к файлам и скриптам существуют физически **(critical)**
- [ACT-2] Referenced файлы (@filename) существуют **(critical)**
- [ACT-3] Версии библиотек и моделей выглядят актуальными **(warning)**
- [ACT-4] Имена переменных, ключей, настроек соответствуют коду **(warning)**

## Как проверять пути (ACT-1)

Если передана структура директории — проверяй каждый упомянутый путь:
- Скрипты запуска: `restart_hub.bat`, `start_hub.pyw`, `voice_input.py`
- Конфигурационные файлы: `voice_config.json`, `ai_config.json`
- Директории модулей: `hub/modules/`, `hub/tools/`

Если структура не передана — помечай как "требует проверки вручную".

## Как проверять @-ссылки (ACT-2)

Найди все строки вида `@filename.md` или `@path/to/file.md`.
Проверь что файл существует в переданной структуре директории.
Несуществующая ссылка — критичная ошибка: агент попытается прочитать файл и получит ошибку.

## Признаки устаревших версий (ACT-3)

Флаги для проверки:
- Gemini Flash / Pro без версии → возможно устарело
- Конкретные версии вида `1.5` у активно развивающихся API → проверить
- VectorBT версия — библиотека редко обновляется, но стоит проверять
- Названия моделей LLM — меняются быстро

## Признаки устаревших имён (ACT-4)

- Ключи в БД / settings: если в CLAUDE.md перечислены ключевые имена настроек —
  они должны совпадать с тем что реально используется в коде
- Имена переменных среды (.env) — должны совпадать с `os.getenv()` вызовами
- Имена MCP-инструментов — должны совпадать с тем что есть в .mcp.json
