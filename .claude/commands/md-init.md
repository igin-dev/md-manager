# /md-init — создание нового MD-файла

Создаёт CLAUDE.md или другой MD-файл (architecture, conventions и т.д.) для проекта.
Анализирует структуру проекта и генерирует правильно оформленный файл с нуля.

## Использование

```
/md-init                           # создать CLAUDE.md в текущей директории
/md-init architecture              # создать ARCHITECTURE.md
/md-init conventions               # создать CONVENTIONS.md
/md-init subproject hub/tools/new  # создать CLAUDE.md для субпроекта (монорепо)
```

## Инструкции агенту

**Шаг 1 — определить тип файла и контекст.**
Разбери аргумент: какой файл создаём и где.
Определи тип проекта: монорепо / обычный репозиторий / субпроект монорепо.

**Шаг 2 — собрать контекст проекта.**
Прочитай (если существуют):
- `package.json`, `pyproject.toml`, `requirements.txt` — стек и зависимости
- `.mcp.json`, `.claude/settings.json` — MCP-серверы и настройки Claude Code
- `README.md` — описание проекта
- Родительский `CLAUDE.md` — если это субпроект (только дельта, не дублировать)
- Структуру директорий (2 уровня)

**Шаг 3 — определить нужные файлы.**
Реши: нужен ли только CLAUDE.md или также ARCHITECTURE.md, CONVENTIONS.md?
Для сложных проектов (>500 строк потенциального контента) — предложи разбивку.
Спроси подтверждение перед созданием нескольких файлов.

**Шаг 4 — запустить субагент md-creator.**
Передай ему весь собранный контекст + тип создаваемого файла + шаблон.

**Шаг 5 — показать результат и запросить подтверждение.**
Покажи черновик. Не записывай файл без явного "да" от пользователя.
