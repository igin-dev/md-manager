# Критерии оптимизации MD-файлов

## Что выносить в отдельный файл

Выносить если секция **самостоятельна** и **не критична для немедленного чтения**:

| Признак | Выносить? | Куда |
|---|---|---|
| Детальная timezone-логика (>15 строк) | Да | CONVENTIONS.md |
| Полный список settings/ключей БД (>10 строк) | Да | CONVENTIONS.md |
| Описание архитектуры компонентов (>20 строк) | Да | ARCHITECTURE.md |
| Схемы потоков данных | Да | ARCHITECTURE.md |
| Соглашения по коду (стиль, именование) | Да | CONVENTIONS.md |
| Критичные запреты | **Нет** | Остаются в CLAUDE.md |
| Gotchas с примерами кода | **Нет** | Остаются в CLAUDE.md |
| Команды запуска | **Нет** | Остаются в CLAUDE.md |

## Что сжимать

**Таблица вместо списка команд:**
```
# Было (6 строк)
Для запуска с выводом: python app.py
Для запуска без консоли: pythonw app.py
После изменений кода: restart_hub.bat

# Стало (4 строки включая заголовок)
| Сценарий | Команда |
|---|---|
| С выводом | `python app.py` |
| Без консоли | `pythonw app.py` |
| После изменений | `restart_hub.bat` |
```

**Убрать вводные слова:**
```
# Было
Важно отметить что SQLAlchemy сохраняет datetime с пробелом.
Следует учитывать что T-формат ломает сортировку.

# Стало
SQLAlchemy сохраняет datetime с пробелом. T-формат ломает сортировку.
```

**Объединить повторяющийся контекст:**
Если одна и та же причина объясняется для нескольких запретов — вынести
причину в заголовок секции, не повторять в каждом пункте.

## Что не трогать

**Никогда не сокращать:**
- Gotcha с примером кода — пример там не случайно, он показывает
  неочевидное поведение которое словами не передать так же точно
- Объяснение причины запрета — без причины запрет теряет половину смысла
- Конкретные значения параметров (maxBytes=5MB, fees=0.001) — они критичны

**Не убирать дублирование если:**
- В дочернем файле уточнение/расширение правила родителя — это не дублирование
- Правило повторяется намеренно для критичного акцента — оставить

## Проверка после оптимизации

После каждого шага оптимизации убедись:
1. Прочитай оригинальный файл
2. Прочитай оптимизированный файл
3. Задай себе: "Потерялась ли хоть одна инструкция агенту?"
4. Если да — вернуть потерянное

Критерий успеха: агент работающий только с оптимизированным файлом
должен принимать те же решения что и агент с оригинальным.
