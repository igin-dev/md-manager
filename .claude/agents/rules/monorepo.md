# Рекомендации: монорепо и иерархия

## Принцип иерархии

Claude Code читает CLAUDE.md снизу вверх по дереву директорий.
Каждый уровень наследует контекст родителя автоматически.

Это значит:
- Глобальный coding standard в корневом CLAUDE.md — не повторяй в дочернем
- Gotcha специфична для одного инструмента — пиши только в его CLAUDE.md
- Одно и то же в родителе и потомке — дублирование

## Что на каком уровне

**Корневой CLAUDE.md (root):**
- Глобальные coding standards (логирование, обработка ошибок, комментарии)
- Карта субпроектов
- Глобальные MCP-серверы
- Общие запреты для всего монорепо

**Субпроект / инструмент:**
- Только дельта от родителя
- Специфичные команды запуска
- Gotchas специфичные для этого компонента
- MCP-серверы специфичные для него

## Признаки дублирования

Дублирование — если в дочернем файле повторяется:
- Тот же coding standard что уже есть в родителе
- То же описание MCP-сервера
- Те же глобальные запреты

Исключение: дочерний уточняет или расширяет правило родителя — это не дублирование.

## Формат карты субпроектов (для root)

```markdown
## Структура

| Директория | Описание | CLAUDE.md |
|---|---|---|
| `hub/` | Flask dashboard :5000 | `hub/CLAUDE.md` |
| `hub/tools/voice-input/` | Whisper voice input | отдельный |
```
