# Правила: размер и плотность

## Чеклист

- [SIZE-1] Файл не длиннее 300 строк **(critical)**
- [SIZE-2] Нет «воды» — каждая строка несёт новую информацию **(warning)**
- [SIZE-3] При >300 строках детали вынесены в referenced файлы через @filename **(warning)**

## Что считается «водой»

- Вступления: "В этом файле описано...", "Важно понимать что..."
- Повторение одного и того же разными словами
- Общие слова без конкретики: "используй лучшие практики", "следуй стандартам"
- Описание очевидного из самого кода

## Что считается правильным выносом в referenced файл

Хорошо выносить:
- Детальные описания алгоритмов (>20 строк)
- Полные таблицы настроек/ключей
- Timezone-логику, форматирование дат — если >15 строк
- Соглашения по коду — в CONVENTIONS.md
- Архитектурные схемы — в ARCHITECTURE.md

Не нужно выносить:
- Критичные запреты (должны быть видны сразу)
- Команды запуска
- Gotchas с примерами кода
