# Правила: структура файла

## Чеклист

- [STR-1] Есть явная секция с командами запуска **(critical)**
- [STR-2] Есть секция со стеком / технологиями / зависимостями **(warning)**
- [STR-3] Для root-файла: есть карта подпроектов с путями к их CLAUDE.md **(warning)**
- [STR-4] Нет дублирования с родительским CLAUDE.md **(warning, если есть родитель)**

## Обязательная структура CLAUDE.md

```
1. Заголовок — одна строка что это и зачем
2. Запуск — конкретные команды, не описание
3. Стек — только то что влияет на решения агента
4. Gotchas / Подводные камни — что неочевидно из кода
5. Запреты — с объяснением причины
6. Ссылки — @ARCHITECTURE.md, @CONVENTIONS.md если есть
```

## Что должно быть в секции "Запуск"

- Конкретные команды bash/cmd, а не описание
- Таблица с сценариями если их несколько
- Указание когда перезапуск нужен / не нужен

Плохо:
```
Проект запускается через Python.
```

Хорошо:
```
| Сценарий | Команда |
|---|---|
| После изменений | `restart.bat` |
| С выводом | `python app.py` |
```

## Что должно быть в секции "Стек"

Только то что влияет на то как агент пишет код:
- Язык и ключевые ограничения версии
- Библиотеки с нестандартным использованием
- Запрещённые альтернативы (например: "только TA-Lib, не VectorBT built-in")

Не нужно:
- Стандартные библиотеки без ограничений
- Всё что есть в requirements.txt если нет специфики

## Карта подпроектов (для root)

Должна содержать путь и ссылку на CLAUDE.md каждого субпроекта:

```
| Директория | Описание | CLAUDE.md |
|---|---|---|
| `hub/` | Flask dashboard :5000 | `hub/CLAUDE.md` |
| `hub/tools/voice-input/` | Whisper voice input | отдельный |
```
