# /md-sync

Синхронизирует CLAUDE.md с актуальным состоянием проекта.

## Использование

```
/md-sync                                         # описать изменения в диалоге
/md-sync добавил MCP-сервер Crypto.com           # описание сразу
/md-sync hub/CLAUDE.md новый тип модуля HYBRID   # конкретный файл + описание
```

## Выполнение

**Шаг 1 — понять что изменилось.**

Разбери `$ARGUMENTS`:
- Есть описание → используй его
- Пусто → спроси у пользователя: "Что изменилось в проекте?"

Типичные поводы: новый MCP, смена библиотеки, новый gotcha, новые соглашения,
новый субпроект в монорепо, изменились команды запуска.

**Шаг 2 — определить затронутый уровень иерархии.**

- Изменение глобальное (coding standard, глобальный MCP) → корневой CLAUDE.md
- Изменение локальное (один инструмент) → только его CLAUDE.md
- Несколько уровней → обновлять каждый без дублирования

**Шаг 3 — собрать контекст.**

Прочитай целевой CLAUDE.md через Read.

Если в репозитории есть git — запусти (кросс-платформенно):
```
git log --oneline -10 || echo "git недоступен"
git diff --stat HEAD~1 || echo "нет предыдущих коммитов"
```

Если изменился MCP — прочитай `.mcp.json` через Read.

**Шаг 4 — подготовить конкретный diff.**

Покажи изменения явно — не описание, а готовый текст:

```
ДОБАВИТЬ в секцию "MCP-серверы":
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
### Crypto.com Exchange MCP
- `get_candlestick` — OHLCV свечи для бэктестинга
- `get_ticker` — текущие цены
Формат символов: `BTC_USDT` (через подчёркивание).
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ЗАМЕНИТЬ в секции "Запуск":
━━━━━━━━━━━━━━━━━━━━━━━━━━━
Было:  `python start.py`
Стало: `python -m uvicorn app:main --reload`
━━━━━━━━━━━━━━━━━━━━━━━━━━━

Обоснование: {почему нужно это изменение}
```

**Шаг 5 — применить и подтвердить.**

Запиши изменения через Write. Коротко подтверди что сделано и покажи итоговый размер файла.

$ARGUMENTS
