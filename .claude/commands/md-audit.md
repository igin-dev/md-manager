# /md-audit — аудит MD-файла по чеклисту качества

Проверяет CLAUDE.md (или другой MD-файл) на соответствие best practice.
Выявляет проблемы, классифицирует по severity, предлагает конкретные правки.

## Использование

```
/md-audit                          # сканирует весь проект, находит все MD-файлы
/md-audit CLAUDE.md                # конкретный файл в текущей директории
/md-audit hub/CLAUDE.md            # файл по пути
/md-audit hub/                     # все MD-файлы в поддиректории
```

## Инструкции агенту

**Шаг 1 — определить область аудита.**
Если аргумент не передан — найди все CLAUDE.md файлы в проекте через glob.
Если передан путь к файлу — работай только с ним.
Если передан путь к директории — найди все MD-файлы внутри.

**Шаг 2 — прочитать контекст иерархии.**
Для каждого файла: прочитай родительские CLAUDE.md (вверх до корня проекта).
Это нужно чтобы выявить дублирование между уровнями.

**Шаг 3 — запустить субагент md-auditor.**
Передай ему:
- Содержимое файла(ов)
- Содержимое родительских файлов (для проверки дублирования)
- Тип файла: root / subproject / tool (определи по расположению)
- Реальную структуру проекта (список файлов в директории)

**Шаг 4 — вывести отчёт.**
Выведи структурированный отчёт от субагента без изменений.
В конце спроси: "Исправить найденные проблемы? Введи /md-fix или укажи какие пункты."
