# /md-audit

Проверяет MD-файлы проекта на соответствие best practice.

## Использование

```
/md-audit                    # все CLAUDE.md в проекте
/md-audit CLAUDE.md          # конкретный файл
/md-audit hub/CLAUDE.md      # файл по пути
/md-audit hub/               # все MD-файлы в поддиректории
```

## Выполнение

**Шаг 1 — определить область аудита.**
Разбери `$ARGUMENTS`:
- Пусто → найди все `CLAUDE.md` в проекте через Glob (`**/CLAUDE.md`)
- Путь к файлу → работай только с ним
- Путь к директории → найди все `*.md` внутри

**Шаг 2 — собрать контекст для каждого файла.**
- Прочитай сам файл
- Найди и прочитай родительские `CLAUDE.md` (вверх до корня) — нужно для проверки дублирования
- Определи тип: `root` (нет родительского CLAUDE.md) / `subproject` / `tool`
- Получи структуру директории файла (`ls` или аналог)

**Шаг 3 — запустить субагент md-auditor.**
Для каждого файла запусти Agent с `subagent_type: "general-purpose"` и промптом:

```
Ты — md-auditor. Прочитай свои инструкции из .claude/agents/md-auditor.md.

Файл для аудита ({тип}):
{содержимое файла}

Родительский CLAUDE.md:
{содержимое или "отсутствует"}

Структура директории:
{список файлов}
```

**Шаг 4 — вывести результат.**
Выведи отчёт субагента без изменений.
Если аудировалось несколько файлов — выводи отчёты последовательно с разделителем.

После отчёта добавь:
```
Для исправления найденных проблем: /md-fix [файл]
```
