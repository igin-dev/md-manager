# /md-sync — обновление CLAUDE.md при изменениях в проекте

Обновляет существующий CLAUDE.md когда в проекте что-то изменилось:
добавлен MCP-сервер, изменился стек, появился новый архитектурный паттерн или gotcha.

## Использование

```
/md-sync                                        # описать изменения в диалоге
/md-sync добавил MCP-сервер Crypto.com          # описание сразу в аргументе
/md-sync hub/CLAUDE.md новый тип модуля HYBRID  # конкретный файл + описание изменения
```

## Инструкции агенту

**Шаг 1 — понять что изменилось.**
Если аргумент содержит описание изменения — используй его.
Если аргумент не передан — спроси: "Что изменилось в проекте?"
Примеры изменений: новый MCP, смена библиотеки, новый gotcha, новый архитектурный паттерн,
новый инструмент в монорепо, изменение соглашений по коду.

**Шаг 2 — определить какой уровень иерархии затронут.**
Изменение касается всего монорепо → обновлять корневой CLAUDE.md.
Изменение касается одного инструмента → обновлять только его CLAUDE.md.
Изменение касается нескольких уровней → обновлять каждый на своём уровне, не дублировать.

**Шаг 3 — прочитать текущее состояние.**
Прочитай целевой CLAUDE.md.
Выполни `git diff --stat HEAD~5` чтобы понять что реально менялось в коде.
Если есть `.mcp.json` — проверь актуальность списка MCP-серверов.

**Шаг 4 — подготовить diff.**
Покажи конкретные изменения в формате:

```
ДОБАВИТЬ в секцию "MCP-серверы":
---
### Crypto.com Exchange MCP
Используй для получения рыночных данных...
---

УДАЛИТЬ из секции "Стек":
- старая строка

ЗАМЕНИТЬ в секции "Подводные камни":
- было: ...
- стало: ...
```

**Шаг 5 — применить только после подтверждения.**
Не вноси изменения без явного "да" от пользователя.
